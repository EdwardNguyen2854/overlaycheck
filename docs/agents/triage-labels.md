# Triage Labels

## Type labels

| Label | Use when |
|-------|----------|
| `feature` | Adding new capability |
| `bug` | Something is broken or wrong |
| `task` | Non-feature work (chore, tooling, docs) |
| `experiment` | Exploring without commitment |

## Area labels

| Label | Use when |
|-------|----------|
| `area:ui` | User interface, components, layout |
| `area:pdf` | PDF loading, rendering, PDF.js integration |
| `area:align` | Alignment controls, position sets |
| `area:review` | Page review, status tracking |
| `area:persist` | localStorage, persistence |
| `area:accessibility` | Keyboard nav, screen reader, a11y |
| `area:build` | Build tooling, CI/CD |

## Priority labels

| Label | Use when |
|-------|----------|
| `priority:high` | Must address before next release |
| `priority:medium` | Should address in next release |
| `priority:low` | Nice to have, no timeline |

## Needs labels

| Label | Use when |
|-------|----------|
| `needs:plan` | Requires a plan before implementation |
| `needs:decision` | Blocked on a user/ stakeholder decision |
| `needs:review` | Needs code review before merge |

## Risk labels

| Label | Use when |
|-------|----------|
| `risk:high` | Could break existing functionality or require significant testing |
| `risk:medium` | Moderate change, some testing needed |
| `risk:low` | Minor change, isolated impact |
